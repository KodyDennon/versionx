# Versionx documentation site

The Versionx user-facing documentation site. Built with Docusaurus 3.

**Live at:** [https://kodydennon.github.io/versionx/](https://kodydennon.github.io/versionx/)

Full contributor documentation (site structure, auto-generators, build + deploy pipeline, how to add pages, MDX gotchas) lives at **[Contributing → Working on the docs](https://kodydennon.github.io/versionx/contributing/documentation)** — or read the source at [`website/docs/contributing/documentation.md`](./docs/contributing/documentation.md).

## Local quickstart

```bash
# One-time setup
npm install

# Dev server with hot reload
npm start                    # http://localhost:3000/versionx/

# Production build
npm run build                # static output → ./build
npm run serve                # serve the built site locally

# Regenerate auto-generated Reference pages
cd .. && cargo xtask docs
```

## Structure at a glance

- **`docs/`** — content, one `.md` file per page.
- **`sidebars.ts`** — sidebar tree; every doc must be referenced here.
- **`docusaurus.config.ts`** — site metadata, plugins, navbar, footer.
- **`src/pages/index.tsx`** — custom landing (not a regular doc page).
- **`src/components/`** — React components used by the landing.
- **`src/css/custom.css`** — Infima brand overrides.
- **`static/`** — favicons, diagrams, images.

Auto-generated pages carry a banner and should never be hand-edited:

```mdx
{/* GENERATED by `cargo xtask docs --only <name>`. Do not edit by hand. */}
```

CI rejects any commit where auto-generated content drifts from source — see [`.github/workflows/docs.yml`](../.github/workflows/docs.yml).

## Deploy

Automatic on every push to `main`. Workflow:

1. `cargo xtask docs` + `git diff --exit-code` (drift gate)
2. `npm ci && npm run build` (build site)
3. `lychee` broken-link check (advisory)
4. `actions/deploy-pages@v4` → GitHub Pages
